# WorkshopLM judge-path interaction rebuild

Date: 2026-07-14

## Verdict

The concept is strong enough to place. The previous interface was not.

The failure was not primarily color, typography, or insufficient OpenAI branding. The interface exposed implementation progress rather than the product thesis. A judge could see cards, drawers, approvals, and outputs, but could not immediately feel the three differentiated interactions:

1. evidence becomes structure;
2. any claim remains reverse-traceable;
3. one approved system produces a coherent package.

The locked direction is **borrow the chrome; invent the interaction**. Use Apps in ChatGPT conventions for shell-level decisions. Preserve WorkshopLM identity in the relational Map, synchronized evidence trace, and output package/storyboard.

## Official Figma kit inspection

Reference: [Apps in ChatGPT · OpenAI Official](https://www.figma.com/community/file/1625636989296445101)

The authenticated Community page was inspected in Chrome. Its own description says “Design for ChatGPT—not screens” and positions the kit as a conversation-first system for inline, actionable apps. The visible page menu exposes:

- Display modes: Inline Card, Inline Carousel, Full screen, Inspector, PiP.
- Core ChatGPT components for Desktop Web, Mobile Web, combined responsive web, iOS, and Android.

The connector could not read the internal layer tree because the Community source does not grant edit access. No internal component names, variables, or layer claims are inferred. Creating an editable copy was not necessary to make the product-level decision.

## What changed in the implementation

### Shell

- Removed persistent Capture → Shape → Deliver navigation.
- Removed generic Workshop Library, generic Details, Outputs shortcut, and browser capture composer from the judge path.
- Reduced the header to Back, current object and purpose, source-scope count, and one contextual primary action.
- Kept ChatGPT as the conversation and voice layer.

### Map

- Removed the visibly broken Excalidraw editor from the judge path.
- Added a constrained spatial grammar: source anchors on the left, grounded claims in the center, implications on the right.
- Added labeled cluster regions and curved provenance edges.
- Selecting a claim dims unrelated content, illuminates its path, and opens one adjacent inspector.
- The inspector combines claim editing, exact evidence excerpt, locator, and a single Open source action.
- Added a real approval summary and named approval receipt.
- Renamed the narrow-screen state to Evidence outline rather than pretending it is a miniature spatial editor.

### Brief and Style

- Replaced the markdown-plus-form impression with a designed approved-brief object.
- Added an approval receipt with source/evidence coverage and downstream consequences.
- Added a live Style contract: palette, typography, three miniature output previews, locked version, and propagation explanation.

### Output package

- Removed the duplicated artifact grid at its source.
- Replaced four equal creation buttons with one Generate/Refresh package action.
- Made deck, infographic, image set, storyboard, and video distinct package objects with Style, Brief, citation, stale, and provider state.
- Preserved the honest boundary: planned image tiles are not represented as generated provider images.

### Storyboard

- Strengthened thumbnails into visual panels with timing and citation state.
- Added selected-panel visual context, narration editing, and source tracing.
- Added a persistent approval summary with panel count, total runtime, grounding coverage, and current dependencies.
- Kept rendering downstream of the approved storyboard only.
